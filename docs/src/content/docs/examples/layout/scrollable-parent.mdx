A common layout puts the canvas inside an `overflow-y: auto` container — for example, a side panel or a fixed-height annotation viewport. Whenever that container scrolls, the canvas moves with it but the pointer event's `clientX/clientY` and `getBoundingClientRect()` both reflect the new viewport position, so the canvas-relative point stays correct.

Scroll the container, then draw. Every stroke lands exactly where the pointer is, including immediately after a scroll.
