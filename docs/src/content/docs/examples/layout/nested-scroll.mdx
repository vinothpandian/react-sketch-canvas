Real layouts often have more than one scroll axis — a horizontally scrolling panel inside a vertically scrolling page, for example. Reading viewport-relative coordinates makes the canvas immune to any number of scrolling ancestors: only the immediate `getBoundingClientRect()` matters, and the browser already accounts for every ancestor when it computes that rectangle.

Scroll the outer container horizontally and the inner container vertically, then draw. Strokes land on the pointer regardless of either scroll position.
