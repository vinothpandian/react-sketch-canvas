## Switching modes

Most drawing tools need at least two modes: adding ink and removing it. `ReactSketchCanvas` exposes that switch through its ref, so your toolbar can stay in React while the canvas handles pointer input.

Keep your UI state in sync with the canvas mode. That lets you show the selected tool, expose the right width control, and avoid confusing users about what the next stroke will do.

## Color selection

The canvas does not prescribe a color picker or toolbar layout. Pass `strokeColor` and `canvasColor` from your own controls, and React will update the drawing surface as those values change.

This pattern works well when color is part of a larger form, theme editor, or annotation workflow.

## Read-only mode

Sometimes a drawing should be visible without being editable: a submitted signature, a saved sketch, or an annotation in review. Use `readOnly` for that state.

Read-only mode keeps the rendered paths on screen while ignoring new pointer input, so you do not need a separate viewer component.
